매개변수의 디폴트 값

함수를 정의 할 때
int MyFunc(int num)
{

}
이런 함수를 호출 할 때 매개변수가 무조건 있어야지만 호출을 할 수 있는 것이다.

매개변수의 디폴트 값이란 이런 것이다
int MyFunc(int num = 10)
{

}
매개변수가 반드시 전달하도록 되어있는데 
만약 매개변수가 들어오지 않는다면 10이 전달 된다.
이런 것이다.

int MyFunc(int num1, int num2)
{

}
이렇게 매개변수가 두개인 경우에도
int MyFunc(int num1 = 1, int num2 = 2)
{

}
매개변수를 전달 하지 않아도 디폴트값으로 들어간다
즉 MyFunc(1,2)와 MyFunc()가 똑같은 것이다.
만약 이런식 MyFunc(3)으로 하나만 전달한다면
MyFunc(3,2)와 똑같은게 되는 것이다.

만약 함수의 선언부와 정의부가 나눠져있다면
선언부에다가 디폴트 값을 넣어야 한다.
왜냐하면 정의부분의 디폴트 값을 넣는다면
컴파일러는 위에서 부터 읽으면서 정의부분에서 이미
디폴트 값이 없었기 때문에 아 애는 디폴트 값이 없구나 하고
내려가서 그 함수를 호출하는 구문을 만나면 
왜 매개변수가 없지? 하면서 에러를 내버린다.

그리고 디폴트 값을 꼭 다채워야 할 필요는 없다
int MyFunc(int num1, int num2 = 2)
{

}
이런 식으로 사용해도 된다.

하지만
int MyFunc(int num1 = 2, int num2)
{

}
이런 식으로는 사용이 안된다
왜냐하면
MyFunc(5)라고 하면
첫번째 매개변수는 무조건 첫번째 매개변수에 들어가게 된다.
이러면 앞에서 num1에 들어가기 때문에
num2가 비어 있기때문에 에러가 나게 된다.

그래서 디폴트 값은 오른쪽에서부터 채워져야 된다.
